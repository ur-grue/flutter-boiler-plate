# /market — front-loaded market research → writes MARKET.md (the research source of truth)

Run this FIRST (before /spec). Fetch live data ONCE, cache it to **MARKET.md**, and never
re-hit Appeeky/mcp-appstore from later steps — they read this file. Input: `APPFACTORY_INPUTS.md`
(idea + category) and AGENTS.md.

## Engines / lanes
- **Live data → `mcp-appstore` tools** (keyless): `search_app` + `get_similar_apps` (5–8 real
  competitors), `get_app_details` + `get_pricing_details` (real monetization **+ each competitor's
  icon AND screenshot URLs — capture the visual/logo style, the screenshot HOOK (first-frame value
  prop), and the color palette; this feeds /theme's icon + palette and /aso's screenshots**),
  `analyze_top_keywords` + `suggest_keywords_by_competition` / `_by_search` / `_by_similarity`
  (candidates), `get_keyword_scores` (real difficulty + traffic), `analyze_reviews` (gaps).
- **Method/routing → `aso-skills`** via `aso-router`: `competitor-analysis`, `market-pulse`,
  `market-movers`, `category-positioning`, `app-marketing-context`, `monetization-strategy`,
  `keyword-research`.
- **Appeeky (optional, if registered):** cross-check first-party data against `mcp-appstore`. Make
  AT MOST ONE Appeeky pass for the whole build — results live in MARKET.md; do not call it again.

## Traceability rule (mandatory)
Every number (keyword difficulty/traffic, competitor price, ranking) MUST carry a source tag —
tool + competitor/term, e.g. `[mcp-appstore:get_keyword_scores]`, `[appeeky]`,
`[mcp-appstore:get_pricing_details / Acme Pro]`. Any figure NOT backed by a live call must be
written verbatim as `LOW-CONFIDENCE (no live data)`. Never fabricate from training knowledge.

## Graceful degradation (headless /mvp may have no web)
1. `mcp-appstore` available → primary. 2. else `appeeky` → use it. 3. else `WebSearch` real
listings (mark every derived figure LOW-CONFIDENCE). 4. else OFFLINE → still write MARKET.md from
idea/category via `aso-skills` method + reasoning, stamp the OFFLINE banner. **Never abort /mvp**
for lack of network — a LOW-CONFIDENCE MARKET.md is still the single source later steps read.

## Write MARKET.md to the repo root with EXACTLY this schema
```
# MARKET.md — market research (generated by /market; the research source of truth)
> DATA MODE: LIVE | PARTIAL (web fallback) | OFFLINE — <one line>
> Generated: <ISO date> · Idea: <from inputs> · Category: <from inputs>

## 1. Competitors & gaps
| App | Store | Primary keywords | Price model | Rating | Gap we can own | Source |
(5–8 real competitors; each row source-tagged)

## 2. Differentiation angle (Apple 4.3)
- One paragraph, meaningfully distinct from §1 (not a clone) + the specific unmet need
  (from analyze_reviews) we build around.
- Visual landscape (feeds /theme + /aso): the common ICON style (colors, glyph vs. wordmark) and
  how ours stands out; the competitors' screenshot HOOK (what value prop they lead with in
  screenshot 1) and dominant COLOR PALETTE — so our screenshots out-hook them and our palette is
  category-credible yet distinct (Apple 4.3).

## 3. Category & positioning
- Primary + secondary store category; where we sit vs. the leaders; why.

## 4. Target keywords (tiered)
### High-traffic (head) / ### Mid / ### Long-tail
| Keyword | Difficulty | Traffic | Source | Notes |
(real get_keyword_scores numbers + source tag, or LOW-CONFIDENCE)

## 5. Pricing (data-driven, from real comps)
- monthly / yearly(-50%) / lifetime anchored to §1 prices; trial norm for the category.
  Each price cites its source competitor.

## 6. Confidence & freshness
- DATA MODE, which engines answered, what is LOW-CONFIDENCE, generated date (for staleness).
```

Then /mvp commits MARKET.md (it is NOT gitignored — it ships with the app repo and is the cache
that /spec, /wire-paywall, and /aso read). Output a 5-line console summary; the file is the deliverable.
